훈련목표 1.1 — 구조적 프로그래밍 설계서를 확인하고 설계 원칙에 따라 프로그램을 설계할 수 있다
문항
구조적 프로그래밍에서는 GOTO문 없이 순차(Sequence)·선택(Selection)·반복(Repetition)의 세 가지 제어 구조만으로 모든 알고리즘을 구현할 수 있다. [  O    /    X  ]
정답
O (참)
해설
빔-야코피니 정리(1966)에 의해 수학적으로 증명된 사실이다. 다익스트라(1968)는 'Go To Statement Considered Harmful'에서 GOTO문의 무분별한 사용을 비판하였으며, 두 이론이 구조적 프로그래밍의 이론적 토대를 형성했다.

80%

훈련목표 2.1 — OOP 설계서를 확인하고 클래스·객체 등 OOP 5대 속성을 설명할 수 있다
문항
객체지향 프로그래밍에서 캡슐화(Encapsulation)란 부모 클래스의 기능을 자식 클래스가 물려받아 확장하는 개념이다. [  O    /    X  ]
정답
X (거짓)
해설
제시된 설명은 상속성(Inheritance)에 해당한다. 캡슐화(Encapsulation)는 속성(데이터)과 행위(메서드)를 하나의 클래스로 묶고, 내부 구현을 외부에 숨겨 데이터를 보호하는 개념이다.

문항
왼쪽의 OOP 속성(①~④)과 오른쪽의 정의((가)~(라))를 바르게 연결하시오.
속성
① 캡슐화
② 다형성
③ 상속성
④ 추상화
정의
(가) 부모 클래스 기능을 자식이 물려받아 확장
(나) 속성·행위를 묶고 내부 구현 은닉
(다) 동일 이름 메서드가 상황별 다른 동작 (오버로딩·오버라이딩)
(라) 공통 특성만 추출하여 정의
정답
①—(나)   ②—(다)   ③—(가)   ④—(라)
해설
캡슐화=데이터 은닉 / 다형성=오버로딩·오버라이딩 / 상속성=부모→자식 계승 / 추상화=공통 특성 추출. 캡슐화↔상속성 혼동이 주요 오답 패턴이다.

문항

② 런타임 오류
③ 논리 오류

문항
컴파일러(Compiler) 방식과 인터프리터(Interpreter) 방식에 관한 설명으로 옳은 것은?
① 인터프리터 방식은 소스 전체를 기계어로 미리 번역하므로 실행 속도가 빠르다
② 컴파일러 방식은 한 줄씩 해석·실행하므로 플랫폼 독립적이다
③ 컴파일러 방식은 실행 파일을 미리 생성하여 실행 속도가 빠르다 ✓
④ Python은 컴파일러 방식의 대표 언어이다
정답
③
해설
컴파일러는 소스코드 전체를 기계어로 번역해 실행 파일을 생성하므로 실행 속도가 빠르다(C, Go, Rust). 인터프리터(Python, JS)는 한 줄씩 해석·실행하므로 플랫폼 독립적이나 속도가 느리다.
